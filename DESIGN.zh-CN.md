# 设计说明

## 设计判断

这是一个给两个人使用的私密日常产品。视觉语言采用 Lo-Fi scrapbook archive：纸张拼贴、可翻面明信片、旧照片、邮戳和克制的状态动效共同构成首屏。

## 方向

视觉锚点是 Lo-Fi。页面应该像一页被认真保存过的拼贴本，藏在一盒旧明信片里：纸黄色底、混合系统字体、轻微旋转的图层、印刷标签、柔和照片颗粒、胶带、邮票、旧卡片边缘和轻微的 risograph 错位。

核心差异点是「可替换的拼贴舞台」。首屏不是静态装饰图。每个装饰图层都有明确类型，Lucky 模式可以替换这些图层，同时保留裁切、纹理、旋转和色彩处理。

## 风格锚点

选定锚点：Lo-Fi。

锚点要求：

- 表面：`#E8E0C0` 或 `#EDE4CF`。
- 字体：混合系统字体，重点使用 Times、Helvetica 和 Courier。
- 结构：元素轻微离网格，旋转角度控制在 2-8 度。
- 纹理：halftone 点阵、纸张颗粒、risograph 错位、胶带、订书钉、撕纸边缘。

这个项目不应漂移到 Swiss 极简、Organic 疗愈风或泛泛的奶油情侣风。页面可以柔和，但仍然需要保留拼贴的张力。

## 情绪

温柔感来自节奏、材料和克制，不来自粉彩和甜腻装饰。

使用：

- 褪色风景照片。
- 有纸纤维感的旧纸。
- 安静的蓝色、邮政红、低饱和橄榄绿、墨色和纸黄色。
- 少量手写质感。
- 缓慢的翻面和替换动效。
- 明信片周围保留足够空纸面，让画面有呼吸感。

避免：

- 把爱心作为视觉系统。
- 贴纸堆叠过多。
- 可爱化微文案。
- 过密的 Pinterest 式拼贴。
- 不承载真实信息的随机英文大字。
- 重阴影、玻璃拟态、霓虹和渐变文字。

## 色彩

使用 Lo-Fi 锚点色作为 CSS 来源 token。

```css
:root {
  --paper: #E8E0C0;
  --paper-lifted: #EDE4CF;
  --ink: #1F1A16;
  --ink-soft: #4E463D;
  --postal-red: #B91F1B;
  --postal-blue: #276A8F;
  --washed-olive: #6C754D;
  --tape: #B98F52;
  --shadow: rgba(31, 26, 22, 0.22);
}
```

主强调色：邮政红。只用于 1-2 个真正有意义的位置，例如日期印章、Lucky 选中状态或明信片邮票细节。不要把红色当作通用装饰色。

辅助强调色：邮政蓝。用于明信片线条、邮戳痕迹和低干扰导航。

## 字体

使用系统字体，以保持 Lo-Fi 锚点。

- 日期：Helvetica 或 Arial Black。字号偏大，通过布局制造压缩感，不依赖夸张 letter-spacing。
- 明信片标签：Courier New 或系统等宽字体。
- 书写内容：后续可以加入高质量且可读的手写字体。第一版先谨慎使用 Times New Roman italic，并用 Courier New 处理结构性明信片文字。
- UI 控件：Helvetica 或 system-ui，保持清楚可读。
- 书写区试验：只在回答书写区域尝试 MingLiU 或 PMingLiU。如果对比度、节奏或移动端可读性不好，回退到 Times New Roman italic。

输入框、标签和控件里不使用展示字体。更有表现力的字体只放在日期、印刷标签和纸片文字上。

## 首屏构图

首屏是一个完整视口高度的拼贴表面，使用 `min-height: 100dvh`。

功能元素：

- 日期对象：放在左上或上中区域，可以处理成印章或大号印刷日期。
- 两张明信片：位于主要视觉区域，可以轻微层叠，但不能被遮挡。
- 过去日期入口：靠近边缘的小标签、票根或邮戳。
- `+` 图标：小型上传入口，用于把个人图片加入当天拼贴。
- 「I'm feeling lucky」：小型工具控件，用于进入替换模式。

装饰元素：

- 两个明信片风景面占位。
- 4-7 个纸片、照片、邮票或胶带元素。
- 最多 1-2 个文字碎片。

构图规则：装饰元素可以相互叠压，但不能遮挡可编辑明信片文本、日期、过去日期入口或 Lucky 控件。

过去日期入口保留在 hero 中。第一版隐藏日期索引。查看过去时直接进入明信片 collection 视图。

## 明信片风景面占位

第一版原型使用两个风景面占位：

1. `postcard-front-a`：山地或海岸风景，偏冷、轻微过曝。
2. `postcard-front-b`：城市或车窗风景，低饱和、胶片感。

两者都是可替换元素，使用同一套处理方式：

- 在明信片边框内使用 3:2 或 4:3 裁切。
- 轻微降低饱和度。
- 使用带旧化边缘的纸质边框。
- 加入轻颗粒。
- 可以加入很小的日期痕迹，但不生成虚假地点，除非用户提供。

## 明信片背面

背面遵循传统明信片结构：

- 顶部标签：`POSTCARD`。
- 左侧书写区：问题和回答字段。
- 右侧地址区：可以保留空的邮寄线，或展示少量保存信息。
- 邮票区：一个小型可替换邮票元素。
- 中间分隔线。

可见字段标签：

- 今天想问你：
- 我的回答：

标签保持朴素。不要把标准 UI 文案改写成主题化说法。

## Lucky 模式

Lucky 模式把表面从浏览状态切换到替换状态。

默认状态：

- 拼贴元素处于被动状态。
- 明信片点击后翻面。
- 过去日期入口保持低存在感。

Lucky 激活状态：

- 可替换元素出现细轮廓或胶带角高亮。
- 功能控件保持可用，但视觉层级低于被选中的元素。
- 选中元素后，在附近打开内联搜索框。
- 搜索结果默认保持原元素类型，除非用户主动切换类型。

替换样式处理：

- 按当前元素框架裁切。
- 继承当前元素的旋转和尺寸。
- 应用项目统一的纸张或照片处理方式。
- 限制饱和度。
- 在不影响可读性的前提下加入轻颗粒或 halftone。

重试行为：

- 重试时在原位置替换候选结果。
- 确认后提交候选结果。
- 取消后恢复上一个元素。

## 上传模式

Hero 中包含一个小型 `+` 图标。两个人都可以上传自己喜欢的图片，并让图片进入当天拼贴。

上传处理：

- 默认裁切成有用片段，而不是完整展示整张图片。
- 轻微降低饱和度。
- 加入纸张边缘或白色撕边。
- 加入轻颗粒。
- 旋转 1-5 度。
- 默认放在明信片后方或旁边，除非用户主动选中调整。
- 原始图片和处理后的视觉图层分开保存。

上传入口应像拼贴表面上的小工具，而不是主按钮。

## 过去 Collection

查看过去时展示一组日期 collage 卡片。一张 collage 卡片代表一个日期。

卡片素材来自当天上传的明信片风景面，也可以混入少量明信片字面碎片。构图使用以下类型之一：

- 胶带拼接：两张明信片风景面轻微重叠，用胶带固定。
- 撕边拼接：两个图片碎片通过白色撕边接合。
- 不规则网格：6-10 个明信片图片碎片组成克制网格，可参考蓝天和黑色形状的 collage ref。
- 风景面和字面混合：一张风景面搭配一小块书写面。

设计判断：

- 第一版优先用胶带拼接或撕边拼接，因为更亲密，也更利于日期识别。
- 不规则网格只在当天图片素材足够强时使用。
- 不要让每张日期卡都变成随机拼贴。每张卡仍然应该像一个完整物件。
- 日期可以作为小型印刷标记出现，不做大标签。

## 动效

动效强度：5/10。

GSAP 只用于真正需要它的位置：

- 明信片翻面：状态转换。
- Lucky 模式元素选中：操作反馈。
- 替换候选元素出现：状态转换。
- 替换后纸片轻微落定：可选反馈。

简单 hover 和 focus 状态优先用 CSS 或 Motion。第一版不使用 GSAP 做滚动叙事。

动效规则：

- 翻面时长：450-650 ms。
- 替换落定：250-400 ms。
- 上传图片出现：250-400 ms。
- 过去 collection 卡片 hover：150-220 ms。
- 缓动：柔和 ease-out，不使用 bounce 或 elastic。
- 只动画 transform 和 opacity。
- 减少动态效果时，翻面改为即时切换加短暂淡入淡出。
- 框架动画必须有生命周期清理。

## 交互状态

明信片：

- 默认：风景面。
- hover 或 focus：轻微抬起，不做明显倾斜。
- active：开始翻面。
- editing：背面保持稳定。
- saved：低干扰保存状态，不依赖频繁 toast。
- error：在相关字段附近显示内联错误。

Lucky 元素：

- passive：不显示额外 UI 外框。
- Lucky active：出现可选中轮廓。
- selected：轮廓加强，并显示内联搜索框。
- loading：使用同形状的纸片占位或局部骨架。
- error：显示朴素的重试提示。

过去日期：

- 使用低存在感标签或入口。
- 打开明信片 collection 视图。
- 第一版隐藏日期索引。

## 响应式行为

桌面：

- 两张明信片可以并排，也可以轻微重叠。
- 拼贴元素围绕明信片制造不对称。

平板：

- 明信片保持居中。
- 装饰元素减少数量，并避开输入区域。

移动端：

- 一次只展示一张主要明信片，并提供清晰的两人切换方式。
- 日期保持可见。
- Lucky 模式仍可选择元素，但搜索框应改为底部面板或固定面板。
- 上传使用系统文件选择器，新图片进入一个可预期的位置。
- 明信片背面的文字不能溢出。

## 素材系统

每个视觉元素都需要元数据：

- `id`
- `type`
- `role`
- `replaceable`
- `position`
- `rotation`
- `scale`
- `zIndex`
- `treatment`
- `source`

功能元素默认锁定。装饰元素和明信片正面可以替换。

上传图片和生成的日期 collage 卡片也使用同一套元数据。后续进入 Lucky 模式时，不需要额外特殊处理。

## 实现说明

- 第一版原型应做成真实组件预览，而不是伪造截图。
- 明信片正面和纸张元素使用真实占位图或生成素材。
- 布局需要 token 化，让替换素材继承统一样式。
- 不依赖无记录的随机绝对定位，需要保存构图状态。
- UI 控件即使嵌入拼贴，也必须保持朴素、清楚、可访问。

## 质量检查

- 隐藏所有装饰元素后，首屏是否仍能使用？
- 两张明信片在移动端是否都能翻面、编辑、保存和阅读？
- Lucky 模式是否清楚展示哪些元素可以替换？
- `+` 上传后的图片是否经过统一视觉处理，并融入页面？
- 过去 collection 是否像一组有日期含义的明信片物件，而不是普通图片墙？
- 替换后的元素是否继承同一套纸张或照片处理？
- 标签是否朴素、清楚、可读？
- 整体是否温柔，但不幼稚、不甜腻、不像模板？
